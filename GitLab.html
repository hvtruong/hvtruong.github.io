GitLab Analyzer

Description

GitLab Analyzer was designed following the Backend For Frontend (BFF) pattern.
It requires accesses to repositories and can analyze them simultaneously. Users can determine the cofigurations like time invervals to analyzer, scores for file types (.css, .html, .java, etc), scores for adding/deleting a line, code reviews, ... Users also receive the code contributions for each commit/ merge request, details of how the scores were computed, and metrics like graphs of code reviews, commit frequencies, ... All these data is stored in our database and available for later uses. Finally, users can adjust the scores based on his/ her decisions and all the correlated scores will be updated automatically.
For this project, I was in the backend team and responsible for designing the API endpoints and the commit/ merge request services. My main work is to maintain the communications between frontend and backed by processing the code contributions, scores, configurations, ..., wrapping them up, and send them to backend or frontend as objects. I also developed a GitLab Automated Continuous Integration System to build and run unit tests whenever anyone pushes code or merges branches.
